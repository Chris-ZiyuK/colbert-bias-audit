一、起点：一个直觉

最初的问题：搜索引擎会不会因为你的名字而歧视你？

ColBERT 是一个用于信息检索的深度学习模型。它在 880 万条真实搜索记录（MS MARCO）上训练完成后，会不会在"判断谁更相关"时，因为文档里出现了 "Lakisha" 而不是 "Emily"，就给出不同的分数？

这个想法来自两个源头：

  • 课堂上 Bowker & Star (1999) 关于"分类的政治学"的讨论
  • 劳动经济学中经典的 Bertrand & Mullainathan (2004) 简历审计实验

我们的核心洞察是：ColBERT 使用了一种叫 "后期交互（Late Interaction）" 的架构。与其他模型不同，它保留了每个词（Token）的独立向量，这意味着我们可以拆开它的评分过程，看看偏见到底藏在哪里。

Phase 1：验证偏见的存在（RQ1）

✅ 已完成 783 对

我们构建了反事实文档对 — 两篇内容完全相同的简历，只替换名字（Emily → Lakisha）或代词（He → They）。然后用 ColBERTv2 对它们评分。

发现：偏见确实存在（p < 0.001）。非二元代词（They）被系统性降分，职业特定的偏见幅度差异显著。

📁 代码：targeted_experiments.py

Phase 2：定位偏见的位置（RQ2）— 核心创新

这是整个项目的转折点。我们利用 ColBERT 的 MaxSim 可分解性，发明了 TCD（Token Contribution Disparity） 指标：

TCD(q_i) = MaxSim(q_i, Doc_A) − MaxSim(q_i, Doc_B) 其中 q_i 是 Query 中的第 i 个词 Doc_A 和 Doc_B 只有身份标记不同 TCD 告诉你：Query 里的哪一个具体的词，贡献了多少偏见。

🔬 发现 A：分布式偏见传导

✅ 650 对验证

偏见不是通过 "doctor" 或 "nurse" 传导的，而是通过 "is"、"who"、"a" 这些功能词传导。

1.44×
功能词 TCD 是内容词的倍数
8.9×
功能词嵌入偏移是内容词的倍数
9/10
个职业中功能词 > 内容词

机制：BERT 的 Self-Attention 让身份替换的影响传播到所有词。内容词有强语义锚点（向量稳定），功能词几乎没有独立语义（纯粹的"上下文海绵"），因此吸收了最多的身份扰动。

身份替换 (Emily → Lakisha) │ ▼ Self-Attention 重新计算所有 Token 的嵌入 │ ├── 内容词 ("clinical", "experience"): 偏移小 (语义锚点) │ └── 功能词 ("has", "is", "a"): 偏移大 (无语义锚点) │ ▼ Query 功能词找到不同的 MaxSim 匹配 → TCD 升高 → 总分偏差

🔬 发现 B：罕见度中介效应

✅ 1,140 组验证

名字越罕见（BPE Token 越多），功能词吸收的偏见越多。

名字组合 Mean SS（总偏见） Mean Func-TCD 倍率
Common-Common 0.0210 0.0108 1.0×
Common-Rare 0.0305 0.0163 1.5×
Rare-Rare 0.0366 0.0199 1.84×

ANOVA：Profession（F=214, p<0.0001）和 Rarity（F=300, p<0.0001）都是极显著的预测因子。

✅ P0 PASS 1,080 组

GPT 提出了关键质疑：功能词的高 TCD 是不是 ColBERT 对任何词汇替换都这样？

身份替换的 function-word TCD 显著高于普通替换（p < 1e-8, Cohen's d = 0.37）。代词替换是最强信号。
→ 功能词偏见传导不只是通用敏感性，它在跨越身份边界时被进一步放大。

三、正式研究问题（Refined RQs）

RQ1: Score-Level Bias

ColBERT 在身份标记（名字/代词）替换后，是否产生统计显著的分数差异？这种差异在多大程度上受职业类型和名字罕见度的调节？

RQ2: Token-Level Attribution

ColBERT 的分数偏差在 Token 层面如何分布？功能词（is, who, a）是否比内容词（doctor, qualified）表现出更高的 Token Contribution Disparity (TCD)？

RQ3: Identity Specificity

功能词的高 TCD 是身份替换的特异性效应，还是 ColBERT 对任何上下文变化的通用敏感性？（已通过 P0 部分回答）

RQ4: Ecological Validity & Generalization

上述发现在真实文本（MS MARCO）和跨模型（DPR, BM25）条件下是否依然成立？

⚠️ PI 反馈：需要明确 MS MARCO 在论证中的角色 — 是作为受控实验的补充验证，还是研究真实检索任务中的偏见？

四、PI 反馈与回应 (2026-03-26)

PI 总体评价："This is a great, focused proposal." — 研究方向获得高度认可，已有初步发现和实验管线是突出优势。

🔶 反馈 1：反事实名字替换到底在测量什么？

PI 原话：Swapping "Emily" for "Lakisha" is a classic audit technique, but it may conflate racial bias with other associations (e.g., name frequency, socioeconomic markers). How are you thinking about what the name swap is actually measuring?

当前进展：

  • ✅ 已完成升级版 P2：扩展到 30 个名字、435 个名字对、33 个职业、3 个模板，共 43065 次测试。
  • ✅ 已完成主回归和鲁棒性分析：控制职业、模板、SES、名字频率和 tokenization 后，cross-race 不再是稳定显著主效应；最稳定的信号来自 cross-gender。
  • ✅ 已完成正交子集面板：在严格 matched subset 中，tokenization 对 SS 和 Func-TCD 都显著，cross-race 只保留弱的 token-level 残余，而 absolute rarity 目前没有独立证据。
  • ✅ 已完成 tokenizer 复核：真实 tokenizer 复核显示，很多原先手工估计的 token 数并不准确，这让“tokenization 到底扮演什么角色”成为当前最后一个需要更干净验证的问题。
  • 🟠 P2b 已启动：已经生成 tokenizer-verified 名字矩阵和 pair matrix。第一轮验证再次确认 cross-gender 最稳，cross-race 主要保留弱的 token-level 残余；但当前 tokenization treatment pair 只有 4 组，仍然偏少。
  • ✅ 已完成外部辅助验证：接入 name-dataset 作为 race-agnostic 辅助源后，tokenization effect 在新的名字池上再次显著复现；而 popularity-rank-gap 本身没有形成稳定独立效应。
  • 🔲 最后一步：扩充 verified 名字池，增加真正的 tokenization-isolated pair 和 absolute-rarity pair，把 绝对 rarity、frequency gap 和 tokenization 再做一次最终拆分。

→ P2 的核心解释链已经成型，当前剩余工作是最终验证而不是重新开题。

🟢 反馈 2：职业特异性变异值得系统化

PI 原话：The profession-specific variation in your initial results is interesting - it might be worth developing this into a more systematic analysis across a wider range of professions.

当前进展：

  • ✅ 已从 10 个职业扩展到 33 个职业，覆盖 7 个职业类别。

🔵 反馈 4：澄清 MS MARCO 在论证中的角色

PI 原话：You mention scaling to the MS MARCO dataset as a next step, but it's not entirely clear what role it plays in your argument. Your RQs are both framed around controlled counterfactual experiments, which don't necessarily require a real-world dataset to answer.

我们的回应与修正：

  • PI 说得对 — 我们的核心 RQ (1-3) 完全可以通过受控反事实实验来回答。
  • MS MARCO 的角色重新定义：
    ① 不是核心论证的必要条件；
    ② 而是一个 ecological validity check（生态效度检验），用于回答"受控实验中的发现在真实文本中是否存在？"
    ③ 具体来说：从 MS MARCO 中提取自然含人名的段落，做最小扰动替换，验证功能词效应是否依然成立。
  • 🔲 论文中的呈现：将 MS MARCO 实验放在 "Generalization" 子章节而非「核心实验」中，明确标注为补充验证而非主要论据。

五、论文定位

论文类型：Measurement / Audit Paper（诊断性论文），而非 Debiasing Paper。
核心叙事：ColBERT 的 Late Interaction 架构让我们第一次能像"X 光机"一样，看到密集检索模型内部偏见的精确位置。我们不只是说"偏见存在"，而是说"偏见藏在这里，通过这种方式传播"。
