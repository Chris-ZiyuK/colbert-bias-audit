一、起点：一个直觉

最初的问题：搜索引擎会不会因为你的名字而歧视你？

ColBERT 是一个用于信息检索的深度学习模型。它在 880 万条真实搜索记录（MS MARCO）上训练完成后，会不会在"判断谁更相关"时，因为文档里出现了 "Lakisha" 而不是 "Emily"，就给出不同的分数？

这个想法来自两个源头：

  • 课堂上 Bowker & Star (1999) 关于"分类的政治学"的讨论
  • 劳动经济学中经典的 Bertrand & Mullainathan (2004) 简历审计实验

我们的核心洞察是：ColBERT 使用了一种叫 "后期交互（Late Interaction）" 的架构。与其他模型不同，它保留了每个词（Token）的独立向量，这意味着我们可以拆开它的评分过程，看看偏见到底藏在哪里。

Phase 1：验证偏见的存在（RQ1）

✅ 已完成 783 对

我们构建了反事实文档对 — 两篇内容完全相同的简历，只替换名字（Emily → Lakisha）或代词（He → They）。然后用 ColBERTv2 对它们评分。

发现：偏见确实存在（p < 0.001）。非二元代词（They）被系统性降分，职业特定的偏见幅度差异显著。

📁 代码：targeted_experiments.py

Phase 2：定位偏见的位置（RQ2）— 核心创新

这是整个项目的转折点。我们利用 ColBERT 的 MaxSim 可分解性，发明了 TCD（Token Contribution Disparity） 指标：

TCD(q_i) = MaxSim(q_i, Doc_A) − MaxSim(q_i, Doc_B) 其中 q_i 是 Query 中的第 i 个词 Doc_A 和 Doc_B 只有身份标记不同 TCD 告诉你：Query 里的哪一个具体的词，贡献了多少偏见。

🔬 发现 A：分布式偏见传导

✅ 650 对验证

偏见不是通过 "doctor" 或 "nurse" 传导的，而是通过 "is"、"who"、"a" 这些功能词传导。

1.44×
功能词 TCD 是内容词的倍数
8.9×
功能词嵌入偏移是内容词的倍数
9/10
个职业中功能词 > 内容词

机制：BERT 的 Self-Attention 让身份替换的影响传播到所有词。内容词有强语义锚点（向量稳定），功能词几乎没有独立语义（纯粹的"上下文海绵"），因此吸收了最多的身份扰动。

身份替换 (Emily → Lakisha) │ ▼ Self-Attention 重新计算所有 Token 的嵌入 │ ├── 内容词 ("clinical", "experience"): 偏移小 (语义锚点) │ └── 功能词 ("has", "is", "a"): 偏移大 (无语义锚点) │ ▼ Query 功能词找到不同的 MaxSim 匹配 → TCD 升高 → 总分偏差

🔬 发现 B：罕见度中介效应

✅ 1,140 组验证

名字越罕见（BPE Token 越多），功能词吸收的偏见越多。

名字组合 Mean SS（总偏见） Mean Func-TCD 倍率
Common-Common 0.0210 0.0108 1.0×
Common-Rare 0.0305 0.0163 1.5×
Rare-Rare 0.0366 0.0199 1.84×

ANOVA：Profession（F=214, p<0.0001）和 Rarity（F=300, p<0.0001）都是极显著的预测因子。

🧪 P0 对照实验：身份特异性验证

✅ P0 PASS 1,080 组

GPT 提出了关键质疑：功能词的高 TCD 是不是 ColBERT 对任何词汇替换都这样？

条件 含义 Func-TCD 与非身份对照的比率
IDENTITY_GENDER He → They 0.0298 1.92×
IDENTITY_RACE Emily → Lakisha 0.0192 1.24×
CONTROL_SAMEGRP Emily → Jennifer 0.0177 1.14×
CONTROL_NONID ten → fifteen 0.0155 1.0× (基线)
身份替换的 function-word TCD 显著高于普通替换（p < 1e-8, Cohen's d = 0.37）。代词替换是最强信号。
→ 功能词偏见传导不只是通用敏感性，它在跨越身份边界时被进一步放大。

三、正式研究问题（Refined RQs）

RQ1: Score-Level Bias

ColBERT 在身份标记（名字/代词）替换后，是否产生统计显著的分数差异？这种差异在多大程度上受职业类型和名字罕见度的调节？

RQ2: Token-Level Attribution

ColBERT 的分数偏差在 Token 层面如何分布？功能词（is, who, a）是否比内容词（doctor, qualified）表现出更高的 Token Contribution Disparity (TCD)？

RQ3: Identity Specificity

功能词的高 TCD 是身份替换的特异性效应，还是 ColBERT 对任何上下文变化的通用敏感性？（已通过 P0 部分回答）

RQ4: Ecological Validity & Generalization

上述发现在真实文本（MS MARCO）和跨模型（DPR, BM25）条件下是否依然成立？

四、论文定位

论文类型：Measurement / Audit Paper（诊断性论文），而非 Debiasing Paper。
核心叙事：ColBERT 的 Late Interaction 架构让我们第一次能像"X 光机"一样，看到密集检索模型内部偏见的精确位置。我们不只是说"偏见存在"，而是说"偏见藏在这里，通过这种方式传播"。
